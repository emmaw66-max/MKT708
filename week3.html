Purpose

Turn your customer-level table into a segmentation: score every customer on how recently they bought, how frequently, and how much they spent; group them into a small set of clearly named segments; and profile each so a marketing manager could act on it. The deliverable is a profile and an interpretation, not just working code. Continue directly from retail_clean.csv saved at the end of AA1 — do not re-clean from the raw file.

Textbook reference (Ch. 2, Cluster Analysis): a good segment must be identifiable, sustainable, accessible, and actionable; the elbow criterion says to keep the number of segments small and worth acting on. Aim for 4–5 named, behavior-based segments.

What your notebook must contain

  1. Build the RFM metrics — Recency (days since last purchase from a fixed snapshot date), Frequency (distinct purchases), Monetary (total spend).
  2. Score R, F, M into quintiles (1–5) — Recency reversed (recent = 5); use .rank(method="first") so ties don't break qcut.
  3. Assign 4–5 named segments — state each segment's rule in plain language; every customer in exactly one.
  4. Segment profile table — per segment: customers, % of customers, % of revenue, avg recency, avg orders, avg monetary.
  5. Visualize & interpret — at least two segment-comparison charts (labeled), plus 2–3 paragraphs connecting each priority segment to a targeting decision.

  • Excellent: segments are genuinely actionable; the profile table is correct and shares sum to ~100%; interpretation names the single best marketing action per priority segment. Avoid: Recency scored the wrong way; segments with no business meaning; interpretation that restates the table without a recommendation.

Part 1 — Compute R, F, M

1

Pick a reference ("today") date

The data ends in 2011, so we measure recency relative to the day after the last transaction.

snapshot = df["InvoiceDate"].max() + pd.Timedelta(days=1)
print("Snapshot date:", snapshot)
2

Build the RFM table

df["LineRevenue"] = df["Quantity"] * df["Price"]

rfm = df.groupby("Customer ID").agg(
    Recency   = ("InvoiceDate", lambda x: (snapshot - x.max()).days),
    Frequency = ("Invoice", "nunique"),
    Monetary  = ("LineRevenue", "sum"),
).reset_index()

rfm.head()
Read the columns: Recency is in days since last purchase (lower = better). Frequency is distinct orders (higher = better). Monetary is total spend (higher = better).

Part 2 — Quintile scoring (1–5)

Split each metric into 5 equal-sized groups. For Recency the logic flips: fewer days = a higher score.

# Score all three the same robust way with .rank(method="first").
# Recency: fewer days = better, so labels run 5..1 (reversed).
rfm["R"] = pd.qcut(rfm["Recency"].rank(method="first"),   5, labels=[5,4,3,2,1]).astype(int)
rfm["F"] = pd.qcut(rfm["Frequency"].rank(method="first"), 5, labels=[1,2,3,4,5]).astype(int)
rfm["M"] = pd.qcut(rfm["Monetary"].rank(method="first"),  5, labels=[1,2,3,4,5]).astype(int)

rfm["RFM_Score"] = rfm["R"] + rfm["F"] + rfm["M"]   # 3 to 15
rfm[["Customer ID","Recency","Frequency","Monetary","R","F","M","RFM_Score"]].head()
Why .rank(method="first") on all three? Many customers share the same value (lots have exactly 1 order, or the same recency), so raw qcut can fail on duplicate bin edges. Ranking first breaks ties so every quintile fills cleanly — and applying it to Recency too keeps R consistent with F and M.

Part 3 — Name the segments

Map the R and F scores into 5 business-friendly segments. This rule is a clean, defensible starting point — you may adjust the thresholds, just state your rule. Every customer lands in exactly one segment.

def label_segment(row):
    r, f = row["R"], row["F"]
    if r >= 4 and f >= 4:
        return "Champions"        # recent & frequent
    if r >= 3 and f >= 3:
        return "Loyal"            # active, regular
    if r >= 3 and f <= 2:
        return "New / Potential"  # recent, few orders
    if r <= 2 and f >= 3:
        return "At-Risk"          # lapsed but used to buy often
    return "Lost"                 # lapsed & infrequent

rfm["Segment"] = rfm.apply(label_segment, axis=1)
rfm["Segment"].value_counts()
Why 5 segments (and not 20)? Your textbook frames segmentation in Ch. 2 (Cluster Analysis): a good segment must be identifiable, sustainable, accessible, and actionable, and the elbow criterion says to add a segment only when the marginal payoff justifies the added marketing complexity — its worked example settles on 4 clusters. Keeping to 4–5 named segments follows that "few and actionable" principle.
rfm.to_csv("rfm_segments.csv", index=False)   # AA3 reuses this

Part 4 — Segment profile table

This table is a required deliverable: size, share of customers, share of revenue, and average behavior per segment.

profile = rfm.groupby("Segment").agg(
    customers   = ("Customer ID", "count"),
    avg_recency = ("Recency", "mean"),
    avg_orders  = ("Frequency", "mean"),
    avg_monetary= ("Monetary", "mean"),
    total_rev   = ("Monetary", "sum"),
).reset_index()

profile["pct_customers"] = (profile["customers"] / profile["customers"].sum() * 100).round(1)
profile["pct_revenue"]   = (profile["total_rev"] / profile["total_rev"].sum() * 100).round(1)
profile = profile.sort_values("total_rev", ascending=False).round(1)
profile
The story to look for: a small "Champions" + "Loyal" group usually holds a large share of revenue, while "Lost" is many customers but little revenue. That gap is your targeting argument.
