Dataset Application Memo

Weight: part of 10%  ·  Due: see Blackboard & syllabus  ·  Submit: PDF

Purpose

Document your first technical interaction with the dataset: confirm you can load it in Python, and show you can read a dataset critically — not just open it, but ask “what is this data, what is missing, and what does that mean?” Every later analysis depends on this dataset, so identifying data-quality issues now is essential.

Required structure (use these headings)

  1. Dataset Overview — rows, columns, column names and types, transaction date range.
  2. Missing Value Analysis — count and % missing per column; interpret what the missingness means for customer-level analysis.
  3. Unique Customer Count — number of unique Customer IDs including, then excluding, missing rows; report the “untrackable” transaction rate.
  4. Initial Data Quality Concerns — at least two issues beyond missing values (cancellations, negative quantities, zero prices) and why they matter.
  5. What This Data Can and Cannot Answer — one marketing question it supports and one it cannot.

Parameters & grading

  • 1–2 pages; must include Python-generated numbers — report the exact output, do not estimate. A small table for the missing-value summary is fine. No code submission required for this memo.
  • File name LastName_FirstName_MKT708_DatasetMemo.pdf  ·  Blackboard → Week 1.
  • Excellent: exact numbers that match Python; missingness interpreted (not just counted); ≥2 quality issues explained; a thoughtful can/cannot-answer section. Avoid: “about 1 million rows”; listing missing values with no implication; skipping the “cannot answer” part.

Deliverable 2 — Dataset Application Memo (1 page · due Jul 13)

Now you load the dataset in Python and report six facts about it. Open a fresh Jupyter notebook in your MKT708 folder and follow along.

1

Load the data

import pandas as pd

df = pd.read_csv("online_retail_II.csv")
df.head()
2

Row count and column types

print("Rows, columns:", df.shape)
print()
df.info()

df.info() prints each column name, its non-null count, and its data type (Dtype). Note which columns are numbers (int64/float64) vs. text (object).

3

Date range

# Convert the date column to real datetimes first
df["InvoiceDate"] = pd.to_datetime(df["InvoiceDate"])

print("Earliest:", df["InvoiceDate"].min())
print("Latest:  ", df["InvoiceDate"].max())
4

Missing values by column

df.isnull().sum()

You'll notice Customer ID and Description have missing values. Make a mental note — those missing Customer IDs become a cleaning decision in Week 2.

5

Number of unique customers

print("Unique customers:", df["Customer ID"].nunique())
6

Pull it together into one report cell

print("=== Dataset Application Memo — Quick Facts ===")
print("Rows:", df.shape[0], "| Columns:", df.shape[1])
print("Date range:", df["InvoiceDate"].min().date(), "to", df["InvoiceDate"].max().date())
print("Unique customers:", df["Customer ID"].nunique())
print("\nColumn types:")
print(df.dtypes)
print("\nMissing values by column:")
print(df.isnull().sum())
The part that earns the grade: after the numbers, write 2–4 sentences flagging initial data quality concerns. Look for: missing Customer IDs (~a few hundred thousand rows), negative Quantity values (returns/cancellations), and zero or negative Price. Name them — don't fix them yet.
# Evidence for your data-quality paragraph
print("Rows with missing Customer ID:", df["Customer ID"].isnull().sum())
print("Rows with Quantity <= 0:", (df["Quantity"] <= 0).sum())
print("Rows with Price <= 0:", (df["Price"] <= 0).sum())
