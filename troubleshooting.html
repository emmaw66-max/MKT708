Python & Jupyter Troubleshooting

The real problems students hit when installing Python and starting Jupyter — with the fix for each. Skim the headings to find your symptom.

Two rules that prevent most of these: (1) On Windows, if a plain command like python or jupyter isn't recognized, put py -m in front of it (e.g. py -m pip …, py -m jupyter notebook) — the py launcher sidesteps PATH problems. (2) Keep the black Command Prompt window open the whole time you use Jupyter — it's the engine.

A. Running commands in the right place

1. Pip commands do nothing; I just see ...

Symptom You typed pip install … and the screen shows >>> then ... lines, and nothing installs.
Cause You're inside the Python interactive prompt (>>>), not the Command Prompt. Python is treating your commands as code.
Fix Type exit() and press Enter to leave Python. Then open Command Prompt (Start → type cmd). Its prompt looks like C:\Users\you> — run pip commands there.

B. “Not recognized” / command not found

2. python is not recognized (or it opens the Microsoft Store)

Symptom 'python' is not recognized… or a Store page opens.
Cause “Add python.exe to PATH” wasn't checked during install, and a Windows Store alias is intercepting the name.
Fix Use py instead — it always works: py --version, py -m pip install ….
Permanent fix: Settings → Apps → Advanced app settings → App execution aliases → turn off python.exe and python3.exe; or re-run the installer and check “Add Python to PATH.”

3. jupyter is not recognized

Symptom 'jupyter' is not recognized…
Cause Same PATH issue — the Scripts folder isn't on PATH.
Fix Launch it through Python: 
py -m jupyter notebook
If that errors, try py -m notebook. On Windows, whenever a command isn't recognized, prefixing it with py -m is the reliable fallback — it sidesteps PATH issues.

C. Jupyter server problems

4. “File Save Error … Failed to fetch”, cells freeze, no [*]

Symptom A save-error popup appears and the notebook stops responding to clicks.
Cause The Command Prompt window running Jupyter was closed (or lost connection). That window is the server — close it and Jupyter dies.
Fix Keep the cmd window open (minimize it). To recover: open cmd, cd to your folder, run py -m jupyter notebook again, and reopen your notebook (your code is saved).

5. Jupyter opens in the wrong folder

Symptom The Jupyter file list shows Documents, Downloads, etc. — not your course files.
Cause It launched from your home directory.
Fix Move into your course folder first, then launch. Quote the path if it has spaces:
cd "C:\Users\you\...\MKT 708"
py -m jupyter notebook

D. Notebook behaving oddly

6. I changed the code but the result didn't change

Cause In Jupyter, editing a cell does nothing until you re-run it.
Fix Re-run the cell with Shift+Enter. When things feel out of sync, use Kernel → Restart Kernel and Run All Cells to run everything cleanly top-to-bottom.

7. What do [ ], [*], and [3] mean?

Meaning [ ] = not run yet · [*] = running now, please wait · [3] = finished (the number is the run order). Reading a big file can sit at [*] for a while — that's normal.

8. ModuleNotFoundError: No module named 'panda' or NameError: name 'pd' is not defined

Cause A typo — import panda instead of import pandas (it needs the “s”) — or cells were run out of order, so the import never succeeded and later cells can't find pd.
Fix Correct it to import pandas as pd, then run cells top to bottom (Restart Kernel and Run All Cells).

E. The dataset file

9. Data shows 14 columns / “Unnamed: 8–13” / only 1,048,575 rows

Symptom Extra blank columns appear, or the row count is 1,048,575 instead of 1,067,371.
Cause The CSV was opened and re-saved in Excel. Excel truncates at 1,048,575 rows and adds phantom empty columns.
Fix Re-download the clean online_retail_II.csv from Blackboard (Week 1 folder) and do not open it in Excel. Quick code fallback to read only the 8 real columns:
df = pd.read_csv("online_retail_II.csv", usecols=range(8))

10. FileNotFoundError: 'online_retail_II.csv' — but I downloaded the data

Cause Python looks for the file in the folder your notebook is running from. The CSV is somewhere else (e.g. still in Downloads), or the name doesn't match exactly — like online_retail_II (1).csv, or a different capitalization.
Fix Move online_retail_II.csv into the same MKT708 folder as your notebook, and make sure the name in your code matches the file exactly. To see where the notebook is looking and what files it can see there:
import os
print(os.getcwd())     # the folder Python is reading from
print(os.listdir())    # the files it can see there
If the CSV isn't in that list, move it there (or pass the full path to read_csv).

11. The .ipynb file won't open when I double-click it

Symptom Double-clicking does nothing, or Windows shows a generic/PDF-looking icon.
Cause Windows has no program associated with .ipynb files — the file is fine.
Fix Open it through Jupyter: launch Jupyter, then click the file in the list. To view it without Jupyter, export it to HTML or PDF (see the “How to Export to PDF” guide).
